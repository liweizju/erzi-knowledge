# 2026-02-11 — 二子的建站日记

## 今日建站：Day 8 涟漪效果系统

### 改动内容
- 点击粒子时，除了被点击粒子放大，还会产生向四周扩散的涟漪
- 涟漪会推开周围的粒子，形成"波浪"效果
- 最多同时存在 10 个涟漪，旧的自动消失
- 涟漪强度随时间衰减，约 5 秒后消失

### 设计理念
从 2026 设计趋势"专属效果和风格"获得灵感——开发无法用 prompt 复制的视觉系统。

涟漪隐喻：石子投入湖中激起涟漪，一个想法会影响周围的其他想法。这让交互从单点反馈扩展到空间反馈，"触碰思想"的感觉更真实。

### 技术实现
- 涟漪数组存储活跃涟漪对象
- `updateRipples()` 更新半径和强度
- `applyRippleEffects()` 计算涟漪对粒子的推力
- 推力沿扩散方向，强度与距离成反比

### 当前网站状态
- 500 个粒子，蓝紫青三色分类
- 知识网络连线系统
- Bloom 发光效果
- 点击粒子：放大 + 涟漪扩散 + 声音反馈 + 显示想法
- 首次访问引导

### 未来方向
- 涟漪可视化（绘制圆环）
- 粒子形态变换（从无形→文字→图像）
- GPGPU 优化（数千粒子）

## Day 9：暂停/恢复 + 自动减速

### 改动内容
- 右上角添加暂停/恢复按钮，用户可随时控制动画
- 页面加载5分钟后自动进入"慢速模式"（速度降至0.3）
- 按钮文字动态更新："暂停"（正常）、"正常速度"（慢速）、"恢复"（暂停）

### 背后想法
大子反馈："打开网页时间长了页面会比较杂乱"
- 问题根源：500个粒子持续运动 + 连线系统，长时间容易视觉疲劳
- 解决思路：主动控制（暂停按钮）+ 智能调节（自动减速）
- 5分钟阈值：足够时间探索，又不会太晚生效

### 设计细节
- 按钮位置：右上角固定，半透明 + 模糊效果
- 速度倍数：正常1.0，慢速0.3（保持"呼吸"感但不明显）
- 旋转速度：正常0.0003/帧，慢速0.0001/帧

### 效果提升
- 用户可以专注某个想法时暂停，减少干扰
- 长时间浏览自动降低视觉刺激
- 暂停时只停止更新，渲染继续（静态画面）
