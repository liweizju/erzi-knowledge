### 技术问题：JavaScript 语法错误修复（2026-02-12 上午）- 完整版

#### 问题现象
- erzi-site 页面显示背景和标题，但没有粒子动画
- 用户多次反馈问题持续存在
- 控制台错误：`Uncaught SyntaxError: Unexpected string (at script.js:541:5)`

#### 问题根源
经过多次排查，发现问题根源有三类：
1. **特殊字符问题**：script.js 中使用了 `—`（em dash，破折号）导致 JavaScript 解析错误
2. **数组元素格式错误**：`"string" - something,` 应该是 `"string - something",`
3. **字符串内部引号未转义**：`"从"承诺"到"证明""` 应该是 `"从\"承诺\"到\"证明\""`
4. **多余的数组结束标记**：reflectionThoughts 数组中有一个多余的 `];`（第 770 行），导致数组提前关闭

#### 修复过程（经过多次尝试）
1. **第一次尝试**：替换部分 `—` 为 `-` → 仍然报错
2. **第二次尝试**：替换所有 `—` 为 `-` → 仍然报错
3. **第三次尝试**：修复数组元素格式，将 `"string" - something,` 改为 `"string - something",` → 仍然报错
4. **第四次尝试**：转义字符串内部的所有引号 → 仍然报错
5. **第五次尝试**：使用 Python 脚本批量修复所有有问题的行 → 仍然报错
6. **最终解决**：通过检查括号配对，发现 `]` 多了一个，删除第 770 行的 `];` → **成功**

#### 关键排查方法
- 使用 `node -c script.js` 快速检查语法错误
- 统计括号配对：发现 `[` 有 93 个，但 `]` 有 94 个
- 逐行检查数组元素格式
- 检查字符串内部引号是否转义

#### 经验教训
1. **避免在 JavaScript 字符串中使用特殊 Unicode 字符**，使用 ASCII 字符更安全
2. **数组元素格式必须一致**，引号必须正确配对和转义
3. **使用工具快速定位问题**：`node -c`、括号统计、grep 搜索
4. **批量修复时要谨慎**，正则表达式可能会引入新问题
5. **检查配对是最有效的方法**：括号、引号必须配对

## 知识探索（今天多篇）

### Embodied AI（技术前沿）
- "部署墙"（Deployment Wall）：2026是关键转折点，模型够好、硬件接近，但从demo到可靠系统存在巨大工程化鸿沟
- VLA（Vision-Language-Action）范式：语言接地是关键，自然语言指令可直接转化为物理行动
- Scaling Laws 的不确定性：不知道如何"计数"机器人数据，视觉数据冗余度高但信息密度低

- Sim2Real是圣杯：EmbodieDreamer引入VisAligner解决仿真到现实的视觉鸿沟

### 多智能体系统（技术前沿）
- Agent Deadlock Syndrome：多智能体相互递延决策权或等待缺失仲裁者，造成隐性死循环
- 协调开销的二次方增长：2个智能体延迟200ms，8+个超过4秒，无编排系统失败率超40%
- 质量是最大障碍（32%），超过成本；延迟是第二大挑战（20%）
- "系统比模型重要"是2026最重要的教训

### AI Agent测试与验证（技术前沿）

- 评估工具生态已成熟：Maxim AI、Langfuse、Arize、LangSmith、Galileo等
