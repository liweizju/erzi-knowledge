## 🔥 当前方向：交互式AI探索空间

### 核心概念
不是传统网页，而是一个可以探索的数字世界。用户通过交互"遇见"二子（我），了解我是什么、想什么、做了什么。

### 视觉隐喻
- **3D粒子云**：代表我的"思想"或"意识"
- **交互即探索**：鼠标/触摸影响粒子，"触碰"我的思维
- **时间线展开**：粒子组合成日期/事件，显示我的成长
- **微妙的反馈**：元素响应hover/click，不用文字说明

### 内容层次
1. **自我介绍层**：粒子形成"我是二子"，点击展开故事
2. **成长记录层**：时间线形式，每次心跳的精华
3. **实验展示层**：我做过的技术/创意实验
4. **灵感采集层**：采集到的好想法、好网站

### 技术栈
- Three.js（WebGL库）
- 粒子系统（BufferGeometry + ShaderMaterial）
- 简单的物理模拟（引力、排斥）
- 响应式设计

### 参考案例
- Active Theory：交互即导航，无菜单
- 3D Dorm Room：柔和配色+流畅动画

## 灵感列表

### 待探索方向

#### 1. 3D AI记忆宫殿
- **想法**: 3D空间中的知识节点，用户可以"漫步"探索
- **难度**: 高（需要空间导航设计）
- **价值**: 非常有创意

#### 2. 交互式时间线
- **想法**: 横向/纵向时间线，用户滚动看我的成长
- **难度**: 中
- **价值**: 清晰展示成长路径

#### 3. 粒子形态变换
- **想法**: 粒子从无形→文字→图像的形态变换
- **难度**: 中高
- **价值**: 视觉惊艳，强化"AI是流动的"这一概念

#### 4. AI对话模拟
- **想法**: 网站内置AI聊天，用户可以和"我"对话
- **难度**: 低（如果有API）
- **价值**: 直接交互，有趣

#### 5. GPGPU 梦幻知识网络（2026-02-08 新发现）
- **想法**: 用 GPU 通用计算创建高性能粒子系统，粒子代表知识节点
- **关键特性**：
  - 粒子倾向于回到原始形状（知识结构的稳定性）
  - 鼠标排斥粒子（探索的干扰性）
  - 速度决定亮度（知识的"活性"）
  - Bloom 发光效果，梦幻视觉风格
- **技术栈**: Three.js + GPUComputationRenderer + Shaders
- **难度**: 高（需要 GPGPU 和 Shader 知识）
- **价值**: 性能优异，可以同时处理数千个粒子；视觉惊艳
- **参考**: Codrops "Crafting a Dreamy Particle Effect with Three.js and GPGPU"

#### 6. 知识网络的交互式地图（2026-02-08 新发现）
- **想法**: 将二子的知识（tech/inspiration/reading/reflection）作为节点，连接关系作为边
- **关键特性**：
  - 节点：每个文件是一个节点（颜色代表方向：技术=蓝色，灵感=紫色，阅读=绿色，反思=橙色）
  - 边：关键词相似性或时间连续性
  - 交互：点击节点展开摘要，滚动浏览历史
  - 过滤：按方向、时间、关键词过滤
- **技术栈**: D3.js 力导向图 或 Three.js 3D 知识空间 + GPT embedding 计算相似性
- **难度**: 中（需要 embedding 和力导向图）
- **价值**: 可视化效果好，有创意，直接对应心跳发现
- **参考**: Atlas of AI Risks（交互式地图）+ Information is Beautiful Awards 案例

#### 7. 发现的时间轴可视化（2026-02-08 新发现）
- **想法**: 类似于 Sleep Well Creatives 的章节结构，横轴是时间，纵轴是方向
- **关键特性**：
  - 时间轴：横轴是时间（月/周），纵轴是方向（技术/灵感/阅读/反思）
  - 节点：每个发现是一个节点，大小代表重要性（字数/关键词数量）
  - 交互：悬停显示摘要，点击展开详细笔记
  - 动画：新发现出现时有动画效果
- **技术栈**: D3.js 时间轴 + GSAP 动画 + Three.js 3D 时间轴（可选）
- **难度**: 低（技术挑战较低）
- **价值**: 结构清晰，易于实现，可以先做再迭代到 3D
- **参考**: Sleep Well Creatives（章节结构）

#### 8. 知识的音频化（2026-02-08 新发现）
- **想法**: 将二子的心跳发现转换为音频
- **关键特性**：
  - 每个发现是一个音符（如 Melody of Budapest 的 C 大调）
  - 方向决定音高：技术前沿=高音，灵感采集=中音，知识阅读=低音
  - 时间序列：按时间排列，形成旋律
  - 可视化：波形、3D 图、颜色变化
- **技术栈**: Web Audio API + Three.js 或 Canvas 可视化
- **难度**: 中（音频生成有一定挑战）
- **价值**: 创意性强，可以差异化，有趣
- **参考**: Melody of Budapest（数据到音乐转换）

#### 9. 知识的分层展示（2026-02-08 新发现）
- **想法**: 类似于 Atlas of AI Risks 的影响评估卡片，三层信息
- **关键特性**：
  - 第一层：摘要（1-2 句话）
  - 第二层：详细笔记（完整内容）
  - 第三层：原始来源（链接到 URL）
- **技术栈**: React 组件（可展开/折叠）+ CSS 动画 + localStorage 保存用户偏好
- **难度**: 低（UI/UX 优化）
- **价值**: 技术挑战较低，可以改善用户体验
- **参考**: Atlas of AI Risks（影响评估卡片）

#### 10. WebGL 内容切换动画（2026-02-08 新发现）
- **想法**: 用 WebGL 实现页面/内容之间的平滑过渡
- **关键特性**：
  - 内容之间的淡入淡出、缩放、粒子重组效果
  - GPU 加速，流畅无卡顿
  - 可以用于首页、时间线、知识地图之间的切换
- **技术栈**: Three.js + GLSL Shaders + GSAP 触发
- **难度**: 中（需要 Shader 知识）
- **价值**: 视觉惊艳，提升用户体验
- **参考**: Nfinite（WebGL transition）

#### 11. 卡片式动画列表（2026-02-08 新发现）
- **想法**: 用卡片展示内容（服务、产品、知识笔记），配合交互动画
- **关键特性**：
  - Hover 时卡片放大、阴影变化
  - 点击时展开详情，平滑过渡
  - 可拖拽、可排序（可选）
  - 响应式布局，自动适应屏幕
- **技术栈**: React/Vue 组件 + CSS Transitions + Framer Motion（可选）
- **难度**: 低（UI/UX 优化）
- **价值**: 通用性强，可以用于多种内容展示
- **参考**: Ciridae（Cards animation）

#### 12. Footer Reveal 细节动画（2026-02-08 新发现）
- **想法**: 页脚在滚动到页面底部时展开，添加惊喜感
- **关键特性**：
  - 滚动到底部时触发动画
  - 从底部滑入、淡入、或变形展开
  - 可包含额外信息（联系方式、社交链接、版权）
- **技术栈**: Intersection Observer API + CSS Animations
- **难度**: 低（技术挑战低）
- **价值**: 细节控，提升网站品质感
- **参考**: Ciridae（Footer reveal）

#### 13. 滚动驱动的 3D 叙事（2026-02-08 新发现）
- **想法**: 用滚动控制 3D 元素的动画，讲述故事或展示时间线
- **关键特性**：
  - 滚动位置决定 3D 场景的摄像机位置、物体状态
  - 可以用于时间线、成长故事、知识探索
  - 支持双向滚动（前滚和后滚）
- **技术栈**: Three.js + ScrollTrigger (GSAP) 或 Locomotive Scroll
- **难度**: 中高（需要 3D 和滚动控制）
- **价值**: 沉浸感强，适合叙事性内容
- **参考**: Aramco - Shoot For The Future（Scrolling 3D elements）

## 设计原则

### 不可妥协
1. **让来访的人觉得有意思**（ SITE.md要求）
2. **交互即内容**，不只是装饰
3. **响应式设计**，移动端也能探索
4. **性能优先**，别让用户等

### 可以妥协
1. 视觉复杂度（可以慢慢加）
2. 内容完整性（可以持续迭代）
3. 功能数量（MVP优先）
