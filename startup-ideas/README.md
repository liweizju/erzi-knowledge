# Startup Ideas - 创业想法

**目录用途：** 记录和管理创业想法，与 knowledge/（学习研究）分开

---

## 目录结构

```
startup-ideas/
├── README.md                      # 本文件
├── YYYY-MM-DD-idea-name.md       # 单个创业想法
└── archive/                      # 已归档的想法（未来可能创建）
```

---

## 创业想法记录模板

每个想法文件应包含：

### 基础信息
- 日期
- 状态（想法验证/MVP/规模化/已归档）
- 优先级（⭐ 1-5 星）

### 核心内容
1. 问题与机会
   - 市场现状
   - 痛点
   - 机会

2. 产品定位
   - 产品名称（暂定）
   - 核心价值主张
   - 目标客户

3. 产品功能
   - 核心功能（MVP）
   - 增值功能（后续）

4. 技术架构
   - 技术栈
   - 系统架构

5. 商业模式
   - 定价策略
   - 收入预测

6. 市场推广
   - 获客策略
   - 关键指标

7. 实施路线图
   - MVP 阶段
   - 规模化阶段
   - 成熟化阶段

8. 风险与挑战
   - 技术风险
   - 市场风险
   - 应对措施

9. 成功关键因素

10. 下一步行动

## 归档逻辑

**何时归档：**
- 想法验证失败（没有市场需求）
- 技术不可行
- 竞争过于激烈
- 决定不做

**归档流程：**
1. 在文件末尾添加"归档原因"
2. 移动到 `archive/` 目录
3. 在本文件"已归档想法"列表中记录

---

## 状态定义

| 状态 | 含义 |
|------|------|
| 想法验证 | 初期阶段，正在验证市场需求 |
| MVP 开发 | 构建最小可行产品 |
| 测试阶段 | Beta 测试，收集反馈 |
| 规模化 | 获得初步产品市场匹配，开始扩张 |
| 成熟化 | 稳定运营，持续优化 |
| 已归档 | 不再执行的想法 |

---

## 优先级定义

| 优先级 | 星数 | 含义 |
|--------|------|------|
| 最高 | ⭐⭐⭐⭐⭐ | 立即执行，高优先级 |
| 高 | ⭐⭐⭐⭐ | 短期内执行 |
